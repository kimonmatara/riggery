riggery.internal.str2api module¶

String representations for Maya API objects.

exception riggery.internal.str2api.Str2ApiBadLookupError¶

Bases: Str2ApiError

Malformed lookup string.

exception riggery.internal.str2api.Str2ApiError¶

Bases: RuntimeError

Unspecified object-retrieval error.

exception riggery.internal.str2api.Str2ApiNoMatchError¶

Bases: Str2ApiError

No scene matches for the specified string lookup.

exception riggery.internal.str2api.Str2ApiTypeError¶

Bases: Str2ApiError

The lookup does not refer to an object of the expected type.

riggery.internal.str2api.getAny(lookup: str, *, firstElem: bool = False, checkShape: bool = True) → tuple[int, tuple[MDagPath | None, MObject] | MPlug]¶
Parameters:

  • lookup – a path to a node, plug or component

  • firstElem – if it’s an attribute, and it’s a multi, return the element at logical index 0; defaults to False

  • checkShape – if lookup refers to an attribute on a node, and the node is a transform, check its shape too; defaults to True

Returns:

A tuple of int (0 for node, 1 for plug, 2 for component) and either a tuple of (MDagPath (or None if not a DAG node) and MObject) if a node or component, or just a single MPlug if an attribute.

riggery.internal.str2api.getArrayContext(mPlug) → tuple[int, MObject] | None¶
Returns:

The index of the array this MPlug belongs to, and the array itself as an MObject.

riggery.internal.str2api.getComponentBundle(lookup: str) → tuple[MDagPath, MObject]¶

Warning

If the lookup is ambiguous, the first match will be returned.

Parameters:

lookup – the path to a component

Raises:

  • Str2ApiNoMatchError – No match for the given lookup.

  • Str2ApiTypeError – The lookup is valid, but doesn’t refer to a component.

Returns:

A tuple of MDagPath and MObject.

riggery.internal.str2api.getMDagPath(lookup: str) → MDagPath¶

Note that, if lookup is a path to a component, a DAG path will be returned the owner node.

Parameters:

lookup – the string representation of the DAG path

Raises:

  • Str2ApiNoMatchError – No match found (or not unique).

  • Str2ApiTypeError – Not a path to a node or component.

Returns:

An MDagPath for the given string lookup.

riggery.internal.str2api.getMPlug(lookup: str, *, firstElem: bool = False, checkShape: bool = True, checkIsPlug: bool = False) → MPlug¶
Parameters:

  • path – the full path to an attribute

  • firstElem – if the attribute is a multi, automatically expand to logical element 0; defaults to False

  • checkShape – if the node is a transform, look for the attribute on its shape too; defaults to True

  • checkIsPlug – Maya will quietly return an invalid MPlug if lookup refers to an existing node (and not an attribute); pass True here to check for this; defaults to False

Returns:

An MPlug reference to the attribute.

riggery.internal.str2api.getMPlugOnMPlug(thisMPlug, attrName)¶
riggery.internal.str2api.getMPlugOnNode(node: MObject, extension: str, firstElem: bool = False, checkShape: bool = True) → MPlug¶

Warning

Doesn’t check if node is a valid MObject.

Parameters:

  • node – the node with the attribute

  • extension – the local path to the attribute (e.g. .input3D[0].input3Dx)

  • firstElem – if the attribute is a multi, automatically expand to logical element 0; defaults to False

  • checkShape – if the node is a transform, look for the attribute on its shape too; defaults to True

Raises:

Str2ApiNoMatchError – The requested attribute couldn’t be retrieved.

Returns:

An MPlug reference to the attribute on the node.

riggery.internal.str2api.getNodeBundle(lookup: str, isDagNode: bool | None = None) → MDagPath | None | MObject¶
Parameters:

  • lookup – an unambiguous path to a DAG or non-DAG node; note that, if lookup instead refers to an existing attribute or component, the owner node will be returned

  • isDagNode – pass True or False here if you already know whether to expect a DAG node, and want to save on a check or throw an exception if it’s not a DAG node; defaults to None

Raises:

  • Str2ApiTypeError – The lookup does not refer to a node.

  • Str2ApiBadLookupError – Wonky lookup string.

  • Str2ApiNoMatchError – No matches found for the lookup (or not unique).

Returns:

Tuple of MDagPath (or None, if not a DAG node) and MObject.

riggery.internal.str2api.getNodeMObject(lookup: str) → MObject¶

Use this instead of getNodeBundle() if you just need the MObject and don’t care about DAG information.

Parameters:

lookup – an unambiguous path to a dependency node; note that, if lookup instead refers to an existing attribute or component, the owner node will be returned

Returns:

A MObject instance for the node.

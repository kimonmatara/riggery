Bases: Tensor

aimTowards(point, aimAxis: str, upAxis: str, upVector=None, *, scale=None, reapplyMagnitudes=True, stretchy=False)¶
Parameters:

  • point – the point towards which to aim; must be an attribute

  • aimAxis – the axis to aim, e.g. ‘x’

  • upAxis – the up axis, e.g. ‘z’

  • upVector/up – if provided, must be an attribute; if omitted, it will be extracted from this matrix; defaults to None

  • scale/s – a scaling factor for all matrix axes; defaults to None

  • reapplyMagnitudes/ram – reuse vector magnitudes from this matrix; defaults to True

  • stretchy/st – make the aim-aligned vector on the output matrix stretch; defaults to False

Returns:

A version of this matrix aimed towards point.

applyViaOpm(*slaves, worldSpace=False, persistentCompensation=False, preserveInheritsTransform=False, maintainOffset=False)¶

Drives one or more transforms via offsetParentMatrix rather than decomposition.

Parameters:

  • *slaves – the transforms to drive

  • worldSpace/ws – drive in world space; defaults to False

  • persistentCompensation/pc – persistently compensate against the SRT channels; defaults to False

  • preserveInheritsTransform/pit – don’t turn off inheritsTransform, detect the parent once instead; defaults to False

  • maintainOffset/mo – preserve initial poses; defaults to False

decompose(*, rotateOrder=None) → dict¶

Convenience method. Pipes this matrix into a decomposeMatrix node and returns a dictionary with these keys: - ‘node’: The decomposeMatrix node - ‘translate’: the ‘outputTranslate’ plug - ‘rotate’: the ‘outputRotate’ plug - ‘quaternion’: the ‘outputQuat’ plug - ‘scale’: the ‘outputScale’ plug - ‘shear’: the ‘outputShear’ plug - ‘rotateOrder’: the ‘inputRotateOrder’ plug

Parameters:

rotateOrder/ro (a standard Maya rotate order enum integer, a rotate order string like 'yxz', or an enum plug) –

if this is None, then: - If this is the ‘matrix’ or ‘xformMatrix’ output on a transform

node, defaults to the node’s ‘rotateOrder’ plug

  • Otherwise, defaults to 0 (‘xyz’)

Decomposes this matrix and uses it to drive one or more transforms.
